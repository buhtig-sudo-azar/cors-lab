# CORS Lab — Уязвимость базовой рефлексии Origin

Учебное веб-приложение по информационной безопасности, демонстрирующее типичную ошибку настройки CORS, при которой сервер автоматически копирует значение заголовка `Origin` из запроса в `Access-Control-Allow-Origin` ответа без проверки.

---

## Описание лабораторной работы

**Тема:** Уязвимость CORS при использовании базовой рефлексии источника

**Цель:** Продемонстрировать начинающим специалистам механизм эксплуатации ошибки конфигурации CORS, при которой любой домен получает доступ к данным авторизованного пользователя.

### Что включено

| Шаг | Описание |
|-----|----------|
| 1 | Базовая структура: React-клиент + API-маршруты |
| 2 | Намеренно уязвимый CORS (рефлексия Origin без проверки) |
| 3 | Имитация авторизованного пользователя с cookie-сессией |
| 4 | Страница атаки `/attacker` — демонстрация эксплуатации |
| 5 | Визуализация HTTP-заголовков (Origin, ACAO, ACAC) |
| 6 | Учебное объяснение «Почему это опасно» |
| 7 | Переключатель «Уязвимый / Безопасный режим» |
| 8 | Таблица сравнения режимов |
| 9 | Практическое задание «Найдите ошибку конфигурации» |
| 10 | Развёртывание на Vercel через GitHub |

## API-маршруты

### GET /api/profile

Возвращает данные профиля пользователя. CORS-политика зависит от текущего режима.

**Ответ:**
```json
{
  "data": {
    "username": "student",
    "email": "student@example.local",
    "role": "user"
  },
  "cors": {
    "mode": "vulnerable",
    "origin": "https://evil-phishing.com",
    "accessControlAllowOrigin": "https://evil-phishing.com",
    "accessControlAllowCredentials": "true",
    "allowed": true,
    "reason": "УЯЗВИМЫЙ РЕЖИМ: ..."
  }
}
```

### GET /api/account

Возвращает конфиденциальные данные аккаунта. Использует учебную cookie-сессию. В безопасном режиме отклоняет запросы с неизвестным Origin.

**Ответ:**
```json
{
  "data": {
    "username": "student",
    "email": "student@example.local",
    "accessLevel": "user",
    "balance": "12 450.00 ₽",
    "accountNumber": "RU-DEMO-0000123456",
    "lastLogin": "2026-06-16T10:30:00Z"
  },
  "cors": { ... }
}
```

### GET /api/mode

Возвращает текущий режим CORS.

### POST /api/mode

Переключает режим CORS.

**Тело:**
```json
{ "mode": "vulnerable" }
```
или
```json
{ "mode": "safe" }
```

## Как работает демонстрация

### Уязвимый режим

1. Сервер читает заголовок `Origin` из запроса
2. Без проверки копирует его в `Access-Control-Allow-Origin`
3. Возвращает `Access-Control-Allow-Credentials: true`
4. **Результат:** Любой домен получает доступ к данным пользователя

### Безопасный режим

1. Сервер проверяет Origin по белому списку доменов
2. Разрешает доступ только доверенным доменам
3. Отклоняет запросы с неизвестным Origin
4. **Результат:** Данные защищены от межсайтового доступа

### Страница атаки (/attacker)

Имитирует вредоносный сайт `evil-phishing.com`, который пытается прочитать данные авторизованного пользователя через кросс-доменный запрос. В уязвимом режиме атака успешна, в безопасном — заблокирована.

## Развёртывание на Vercel через GitHub

### Шаг 1: Создание репозитория на GitHub

```bash
# Инициализация Git (если ещё не инициализирован)
git init

# Добавление файлов
git add .

# Создание коммита
git commit -m "feat: CORS Lab — учебный стенд по информационной безопасности"

# Добавление удалённого репозитория
git remote add origin <URL-репозитория>

# Отправка в GitHub
git push -u origin main
```

### Шаг 2: Подключение к Vercel

1. Перейдите на [vercel.com](https://vercel.com) и авторизуйтесь
2. Нажмите **«Add New»** → **«Project»**
3. Выберите **«Import Git Repository»**
4. Найдите и выберите ваш репозиторий на GitHub
5. Нажмите **«Import»**

### Шаг 3: Настройка проекта

Vercel автоматически определит Next.js и предложит настройки по умолчанию:

- **Framework Preset:** Next.js
- **Root Directory:** `./`
- **Build Command:** `next build`
- **Output Directory:** `.next`

Нажмите **«Deploy»** и дождитесь завершения.

### Шаг 4: Обновление белого списка

После развёртывания добавьте URL вашего приложения в белый список в файле `src/lib/cors.ts`:

```typescript
const ALLOWED_ORIGINS: string[] = [
  "http://localhost:3000",
  "https://ваш-проект.vercel.app",  // Добавьте ваш URL
];
```

### Шаг 5: Автоматический деплой

После подключения Vercel автоматически развёртывает приложение при каждом push в основную ветку:

```bash
git add .
git commit -m "fix: обновление белого списка доменов"
git push
```
